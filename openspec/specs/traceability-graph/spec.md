## Requirements

### Requirement: Bidirectional graph of artifacts

The traceability service SHALL persist a graph of nodes (`openspec`, `jira_issue`, `jira_epic`, `confluence_page`, `adr`, `pr`, `commit`, `deployment`, `test_run`, `slo`, `incident`, `cost_record`) and links (`derives_from`, `implements`, `validates`, `deploys`, `monitored_by`, `cost_attributed_to`).

#### Scenario: Link created when PR references OpenSpec

- **GIVEN** PR #42 references `OpenSpec: spec-7`
- **WHEN** the traceability service receives `pr.linked-to-openspec.v1`
- **THEN** a node `pr/#42` MUST be created if absent
- **AND** a link `(pr/#42) -[implements]-> (openspec/spec-7)` MUST be created
- **AND** emit `traceability.link.created.v1`

#### Scenario: Link created on deployment

- **GIVEN** deployment `dep-9` for asset `app-foo` references `spec-7`
- **WHEN** `deployment.applied.v1` is consumed
- **THEN** node `deployment/dep-9` MUST be created
- **AND** link `(deployment/dep-9) -[deploys]-> (asset/app-foo)`
- **AND** link `(deployment/dep-9) -[derives_from]-> (openspec/spec-7)`

### Requirement: Query API by OpenSpec root

`GET /v1/traceability/{openspec_id}` MUST return the materialized subgraph rooted at the OpenSpec with all related nodes within N hops (default 4).

#### Scenario: Auditor traces full path

- **GIVEN** `spec-7` linked through architecture → design → PRs → deploys → SLOs → incidents
- **WHEN** `GET /v1/traceability/spec-7?depth=4` is called
- **THEN** the response MUST contain all nodes reachable in 4 hops
- **AND** each link MUST include relation type, source event, timestamp

### Requirement: Backfill from history

The service MUST be able to backfill the graph from existing audit logs and events generated by Phases 1–3 prior to the service going live.

#### Scenario: Backfill runs successfully

- **GIVEN** audit logs containing 1000 historical events
- **WHEN** the backfill job runs
- **THEN** it MUST create nodes/links for each relevant event
- **AND** complete within an acceptable window
- **AND** emit `traceability.backfill.completed.v1` with counts

### Requirement: Materialized views for performance

Query endpoints MUST be backed by materialized views refreshed at most every 5 minutes; query p95 latency MUST be ≤ 1s.

#### Scenario: Performance under load

- **GIVEN** a traceability graph of 100k nodes
- **WHEN** 100 concurrent queries hit `GET /v1/traceability/{id}`
- **THEN** p95 latency MUST stay ≤ 1s
- **AND** materialized view freshness MUST be ≤ 5 minutes
