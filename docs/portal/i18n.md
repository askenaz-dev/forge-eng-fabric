# Forge Portal — Internationalisation (ES / EN)

The Portal ships with two supported locales: **Spanish (default)** and
**English**. Every visible string in shipped Portal code MUST come from the
dictionary at `portal/src/i18n/dictionary.ts`. Hard-coded English or Spanish
literals in component source are forbidden and flagged in code review.

## Adding a new key

1. Open `portal/src/i18n/dictionary.ts`.
2. Add the key to **both** the `es` and `en` blocks. Keys are kebab- or
   underscore-cased (`apr_approve`, `nav_dashboard`, `cmd_action_lang_es`).
3. If the string contains a variable, use `{name}` placeholders:
   ```ts
   ev_run_started: "El agente {agent} inició un run en {repo}",
   ```
4. Run `pnpm --filter @forge/portal run i18n:check` — the parity script
   exits non-zero if ES and EN diverge.
5. If the term is part of the platform glossary (below), also update the
   `glossary-audit.mjs` PAIRS list to lock the canonical translation.

## Using `t()`

```tsx
"use client";
import { useLang } from "@/components/providers/LangProvider";

export function GreetUser() {
  const { t } = useLang();
  return <h2>{t("h_hello_pm")} Ana</h2>;
}
```

Variable substitution:

```tsx
const { t } = useLang();
const txt = t("ev_apr_granted", { target: "orders-api", who: "ana@acme.io" });
// → "Aprobado el cambio sobre orders-api por ana@acme.io"
```

## Date, number and duration formatting

Use the helpers in `portal/src/i18n/format.ts` — they wrap `Intl` and
cache formatters:

- `formatNumber(lang, value, options?)`
- `formatDate(lang, ts, options?)`
- `formatRelativeTime(lang, ts, now?)`
- `formatDuration(seconds)` — always `mm:ss`

## Initial paint

Server render of `layout.tsx` reads `forge_prefs.lang` from the cookie and
sets `<html lang>` so the first byte already carries the correct locale.
The `LangProvider` hydrates with that initial value, persists subsequent
changes to `localStorage`, and POSTs to `/api/i18n/preference` which
mirrors the choice to `users.preferences.locale` in the control-plane.

## Pluralisation

`t()` accepts `{ count }` for pluralisation. Today the dictionary uses
simple substitution; if you need full ICU MessageFormat (e.g. `{count,
plural, one {# run} other {# runs}}`), wrap with `Intl.PluralRules` at
call-site and pick the matching key.
