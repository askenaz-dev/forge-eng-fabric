# Evolution Loop

The evolution loop converts postmortem learnings into OpenSpec change
proposals. Implemented by `services/evolution` (Go).

## Trigger

`postmortem.published.v1` fires the loop. The skill `derive-openspec-evolution`
analyses the postmortem and produces a `Proposal` containing:

- `acceptance_criteria` derived from each lesson;
- a `runbook_update` listing the healing actions used;
- a `new_gate` if severity was `critical`;
- a `new_healing_action` if no healing action existed in the catalog.

## Source marker

Every proposal generated by the loop carries `source = "autonomous-loop"`.
When the proposal is converted to an OpenSpec change, the OpenSpec backbone
preserves this marker and the Portal shows a distinct **autonomous-loop**
badge in the Evolution Inbox.

## Status lifecycle

```
draft → inbox → accepted → converted
              ↘ rejected
```

- **inbox** — created by the evolution service; awaiting human review.
- **accepted** — reviewer approved; OpenSpec change creation pending.
- **converted** — converted to a normal OpenSpec change.
- **rejected** — reviewer dismissed (with optional comment).

## Endpoints

```
POST /v1/evolution/from-postmortem    — create a proposal from a postmortem
GET  /v1/evolution/proposals          — list (filter by status, tenant)
GET  /v1/evolution/proposals/{id}     — fetch one
POST /v1/evolution/proposals/{id}/review
                                       — apply review decision
GET  /v1/evolution/stats              — counts + acceptance ratio
```

## Metrics

`services/evolution` exposes Prometheus counters:

- `evolution_proposals_total{status}`
- `evolution_proposals_acceptance_ratio` (gauge — exported by the stats endpoint)

Both feed the "Phase 6 — Autonomous Ops" Grafana dashboard.

## Human review

Proposals from the autonomous loop **never** become normal OpenSpec changes
without explicit human approval. The OpenSpec backbone enforces this with the
`source` field and the new `review_status` lifecycle (see
`services/openspec/openspec_service/store.py:review`).

## Audit

Every transition emits `evolution.openspec_proposal.v1` and
`openspec.autonomous_loop.reviewed.v1` for traceability.
